1ο παραδοτέο - SRS

Παραδοχές:
  Με τον όρο δραστηριότητα(event) εννοούμε την κάθε δραστηριότητα που φτιάχνει ένας provider, εγκρίνει ένας διαχειριστής και δείχνει ενδιαφέρων ένας parent. Συμπεριλαμβάνει τίτλο, τοποθεσία, ημερομήνια και ώρα, περιγραφή ελευθέρου κείμενο και εικόνες για την δραστηριότητα του.
  Υπάρχουν λέξεις με " " τις οποίες αναλύουμε στο κάτω μέρος της σελίδας μαζί με κάποιες άλλες σχεδιάστηκες επιλογές.


Anonymous user:
  Θα μπορεί να κάνει "εγγραφή" ως parent ή ως provider
  Θα μπορεί να κάνει είσοδο με τα στοιχεία του λογαριασμού του
  Θα μπορεί να δει όλες τις δραστηριότητες που παρέχουν όλοι οι providers
  θα μπορεί να "αναζητήσει" δραστηριότητες με χρήση ελεύθερου κειμένου
  θα μπορεί να "αναζητήσει" δραστηριότητες με χρήση φίλτρων (για ηλικία, είδος δραστηριότητας, τιμή, τοποθεσία)
  θα μπορεί να "αναζητήσει" δραστηριότητες συνδυάζοντας όλα τα παραπάνω
  θα μπορεί να "επιλέξει" μια δραστηριότητα για να δει περισσότερες πληροφορίες για αυτήν 

Parent user:
  Θα μπορεί να κάνει έξοδο από το λογαριασμό του
  Θα μπορεί να δει όλες τις δραστηριότητες που παρέχουν όλοι οι providers
  θα μπορεί να "αναζητήσει" δραστηριότητες με χρήση ελεύθερου κειμένου
  θα μπορεί να "αναζητήσει" δραστηριότητες με χρήση φίλτρων (για ηλικία, είδος δραστηριότητας, τιμή, τοποθεσία)
  θα μπορεί να "αναζητήσει" δραστηριότητες συνδυάζοντας όλα τα παραπάνω
  θα μπορεί να "διαχειρίζεται" την πιστωτική κάρτα
  Θα μπορεί να "διαχειρίζεται" τα στοιχεία του
  θα μπορεί να "φορτίσει" το πορτοφόλι του είτε κάνοντας χρήση μια πιστ. κάρτας από τις υφιστάμενες, είτε εισάγοντας μια νέα
  θα μπορεί να "επιλέξει" μια δραστηριότητα για να δει περισσότερες πληροφορίες για αυτήν 
  θα μπορεί να "αγοράσει" εισιτήριο για την δραστηριότητα που επέλεξε και το εισιτήριο θα αποθηκεύεται αυτόματα
  θα μπορεί να βλέπει το ιστορικό πληρωμών του
  θα μπορεί από το ιστορικό πληρωμών του να δεί όλα τα εισιτήρια που έχει πληρώσει

Provider user:
  Θα μπορεί να κάνει έξοδο από το λογαριασμό του
  Θα μπορεί να "διαχειρίζεται" όλες τις δραστηριότητες του
  Θα μπορεί να "διαχειρίζεται" τα στοιχεία του

Admin user:
  Θα μπορεί να κάνει έξοδο από το λογαριασμό του
  Θα μπορεί να "διαχειρίζεται" όλες τις δραστηριότητες των providers
  Θα μπορεί να "διαχειρίζεται" τα στοιχεία του
  Θα μπορεί να "διαχειρίζεται" τους λογαριασμούς των χρηστών
  Θα μπορεί να "αναζητήσει" τους χρήστες, δραστηριότητες


Για τον Anonymous user:
  "εγγραφή":Να δημιουργήσει λογαριασμό με όλα τα απαραίτητα στοιχεία και να επιλέξει αν είναι provider ή parent και σε
    κάθε περίπτωση προστίθενται τα αντίστοιχα υποχρεωτικά πεδία για κάθε ρόλο.
  "αναζητήσει": Να βρει δραστηριότητες μέσω φίλτρων/keywords.
  "επιλέξει":Να πατήσει σε μία δραστηριότητα για να δει περισσότερες πληροφορίες.

Για τον Parent user:
  "αναζητήσει":Να βρει δραστηριότητες μέσω φίλτρων/keywords.
  "διαχειρίζεται":Να αφαιρέσει, προσθέσει ή επεξεργαστεί τις πιστωτικές του κάρτες ή τα προσωπικά του στοιχεία
  "φορτίσει":Να προσθέσει χρήματα στο ηλεκτρονικό πορτοφόρλι πληρώνοντας με τις προσωπικές του κάρτες
  "επιλέξει":Να πατήσει σε μία δραστηριότητα για να δει περισσότερες πληροφορίες.
  "αγοράσει":Να χρησιμοποιήσει τα χρήματα που υπάρχουν στο ηλεκτρονικό πορτοφόλι για να αγοράσει ένα εισιτήριο.

Για τον Provider user:
  "διαχειρίζεται":Να προσθέτει να αφαιρεί ή να επεξεργάζεται τις δραστηριότητες που έχει δημιουργήσει καθώς να τροποποιεί τα στοιχεία του

Για τον Admin user:
  "διαχειρίζεται":Να αφαιρεί ή να επεξεργάζεται τις δραστηριότητες της εφαρμογής, να τροποποιεί τα στοιχεία του, να διαχειρίζεται και να
    ελέγχει τους χρήστες(Ανάθεση και ανάκληση ρόλων και δικαιωμάτων, Κλείδωμα χρήστη, Reset password χρήστη)
  "αναζητήσει":Να βρει χρήστες και δραστηριότητες μέσω φίλτρων/keywords 


Γενικές προδιαγραφές:
  Η σελίδα προσθέτει ένα 5% στην τιμή κάθε εισιτηρίου για το δικό της κέρδος. 
  Οι χρήστες, εκτός και αν κάνουν logout, παραμένουν συνδεδεμένοι με την χρήση cookies αυτόματα και δεν πρέπει να το επιλέξουν εκείνοι.
  Για να φορτώσει το πορτοφόλι του ο χρήστης για τους λόγους της εργασίας απλά προσθέτει μια κάρτα(τα νούμερα κλπ) και απλά επιλέγει πόσα χρήματα να προσθέσει. Δεν γίνεται χρήση κάποιου portal.
  Οι providers δίνουν κατά την εγγραφή(ή και στο profile page) ένα IBAN το οποίο με κάθε αγορά εισιτηρίου τα χρήματα πηγαίνουν εκεί αυτόματα και απλά εμφανίζεται σαν στατιστικό το πόσα έχει κερδίσει ο provider.
  Για τα φίλτρα:
    Αρχικά οι δραστηριότητες εμφανίζονται με φθίνουσα σειρά ανάλογα με την ημερομήνια δημιουργίας τους. Δίνεται στον χρήστη η δυνατότητα μέσω φίλτρων να περιορίσει τα events που του εμφανίζονται ήδη, αλλά και με την χρήση keywords να εμφανίσει συγκεκριμένα events. Επίσης δίνεται η δεινότητα συνδυασμού των φίλτρων με τα keywords, σε περίπτωση που ο χρήστης έχει γράψει κάποιο γενικό keyword και θέλει να χρησιμοποιήσει τα φίλτρα για να περιορίσει την αναζήτηση του. Τα φίλτρα επίσης μόνα τους μπορούν να συνδυαστούν μεταξύ τους χωρίς να είναι υποχρεωτική η χρήση keywords.
    Συγκεκριμένα, τα φίλτρα βρίσκονται σε 3 κατηγορίες,
    Το πρώτο φίλτρο δίνει την δυνατότητα στον χρήστη να δει για events που γίνονται "σήμερα" ή "αυτή την εβδομάδα".
    Το δεύτερο φίλτρο δίνει την δυνατότητα στον χρήστη να περιορίσει την αναζήτηση σε κάποια συγκεκριμένη περιοχή.
    Το τρίτο φιλτρο περιορίζει την αναζήτηση σε δραστηριότητες εξωτικού ή εσωτερικού χώρου.
  Αφού δημιουργήσει ένα event ο provider, αυτό το event περνάει από έλεγχο από τον διαχειριστή, το επιθεωρεί και το εγκρίνει ή το απορρίπτει. Μέχρι να παρθεί μια απόφαση από τον διαχειριστή δεν εμφανίζεται δημοσία πάρα μόνο στις διαχειριστικές σελίδες του διαχειριστή και του provider που το δημιούργησε.
